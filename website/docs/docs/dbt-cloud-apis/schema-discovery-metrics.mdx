Below we show some illustrative example queries and outline the schema (all possible fields you can query) of the metrics object.

### Example query

The example query returns information about all metrics for the given job.

```graphql
{
  metrics(jobId: 123) {
    uniqueId
    name
    packageName
    tags
    label
    runId
    description
    type
    sql
    timestamp
    timeGrains
    dimensions
    meta
    resourceType
    filters {
       field
       operator
       value
     }
    model {
      name
    }
  }
}

```

### Fields
The metrics object can access the _same fields_ as the [metric node](/docs/dbt-cloud-apis/discovery-schema-metric). The difference is that the metrics object can output a list so instead of querying for fields for one specific metric, you can query for those parameters for all metrics in a run.
