When querying for `seeds`, the following arguments are available. If you don't include a runId, the API defaults to the most recent run of the specified job:

Below we show some illustrative example queries and outline the schema of the seeds object.

### Example query

The query below pulls relevant information about all seeds in a given job.  For instance, you can view load times.

```graphql
{
  seeds(jobId: 123) {
    uniqueId
    name
    executionTime
    status
  }
}
```
